## Post (EN, da Dev-Audience international)
Spent a weekend turning my best agent prompts into reusable, documented "skills" instead of
re-pasting the same 200-line prompt every session.

What actually moved the needle:
- One skill = one job, with a strict INPUT/OUTPUT contract (no "do everything").
- Examples + edge cases in the skill, not in my head.
- A short "verification" step so the agent checks its own output before I read it.

If you build agents: packaging your prompts as skills is the highest-ROI cleanup you'll do.

I bundled 6 of mine (code-review, debugging, TDD, planning, research, skill-authoring) as a
starter pack — link below. Not a course, just the files.
